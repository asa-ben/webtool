# tobila-call-relinker 定期実行セットアップガイド

## 仕組みの選択：Claude Code クラウドScheduled Task（Routine）

2026年3月から、Claude Codeに**クラウド型の定期実行（Routines / Scheduled Tasks）**が追加されています。

- 実行場所は**Anthropicのクラウド**。福井先生のPCが閉じていても動く
- Pro/Max/**Team**プランで利用可能（弊所のTeam Standardで利用可）
- claude.aiで接続済みの**コネクタ（Salesforce・Gmail・Google Drive）をそのまま利用可能**（Routine作成時にデフォルトで全コネクタが含まれる）
- 最短実行間隔は1時間。日次実行は問題なし
- 各実行は毎回フレッシュなセッションで開始されるため、**プロンプトは自己完結**している必要がある

ローカルのDesktop scheduled tasksや/loopは**PCが起動していないと動かない**ため、今回の要件（PC停止中の実行）にはクラウド版一択です。

## セットアップ手順

### 1. スキルをGitHubリポジトリに配置

Routineはリポジトリをクローンして実行されるため、スキル本体をwebtoolリポジトリに置きます：

```
（webtoolリポジトリ）
└── .claude/
    └── skills/
        └── tobila-call-relinker/
            └── SKILL.md   ← 全弁護士対応版
```

こうすると、スキルの修正はリポジトリへのpush一回で翌日の実行から反映されます（Routine本体の再設定不要）。

### 2. Routineの作成

claude.ai/code/scheduled を開き「New scheduled task」、またはClaude Code CLIで `/schedule` コマンド。設定値：

| 項目 | 設定 |
|---|---|
| 名前 | tobila-call-relinker-nightly |
| リポジトリ | webtool（.claude/skills/ にスキルを置いたリポジトリ） |
| スケジュール | 毎日 06:00 JST（cron: `0 6 * * *`）※前日分を朝の始業前に処理 |
| コネクタ | **salesforce と Gmail のみ残し、他は外す**（最小権限。Drive はロールバックCSV保存に使う場合のみ残す） |
| プロンプト | 下記 |

### 3. Routineプロンプト（コピペ用）

```
このタスクはサブエージェントに全作業を委任して実行すること。まずサブエージェントを1つ起動し、以下をすべてサブエージェント内で実行させる（メインセッションでは作業しない）。

サブエージェントへの指示：
リポジトリ内の .claude/skills/tobila-call-relinker/SKILL.md を読み、その手順に厳密に従って「無人実行モード」で実行する。

- 対象期間：昨日（日本時間）の1日分
- 対象弁護士：SKILL.mdの対象弁護士マスタに記載の全員
- 高確度・中確度（SKILL.md Step 6の基準）の架電はWhatIdを自動更新する。判定不能・候補なしは更新せず保留リストに回す（手動紐付け対象として報告する）
- 更新前に必ずロールバックCSVを作成する
- 処理完了後、SKILL.md Step 11に従い、Gmailで (a)福井先生（fukui@a-fukui-law.com）宛の管理者報告メールを1通、(b)中確度の自動紐付けまたは保留分がある各弁護士宛に、本人分のみを記載した確認通知メールを1通ずつ送信する
- Salesforceへの書き込みはTaskのWhatId更新のみ。それ以外のオブジェクトは読み取り専用で扱うこと
- 処理対象が0件の場合も、福井先生宛に「本日0件」のメールを1通送ること（実行確認のため。この場合、各弁護士宛のメールは不要）
```

> **サブエージェント委任にしている理由：** クラウドScheduled TaskでMCPコネクタのツールがメインセッションに読み込まれない既知の不具合（GitHub Issue #43397）があり、サブエージェントに委任するとコネクタが正常に初期化されるワークアラウンドが確認されています。修正され次第、冒頭の委任指示は外して構いません。

### 4. 初回はドライランで検証

最初の1〜2回は、プロンプトの「高確度・中確度の架電はWhatIdを自動更新する」を一時的に
「**WhatIdの更新は一切行わず、更新予定リストをメール報告するだけにする（ドライラン）**」
に差し替えて実行し、報告メールの判定精度を確認してから本番化することを推奨します（SyncFolderNames.gsのDRY_RUN運用と同じ考え方）。

※2026/6/11にチャット上で対話モードの試運転を実施済み（前日分35件、高確度11件を紐付け実行・検証OK、中確度21件も判定は概ね正確と確認）。この結果を受けて中確度の自動紐付けを有効化しています。

## 運用イメージ

1. 毎朝6:00、クラウドで前日分の架電を自動処理
2. 高確度・中確度分は自動で基本情報に紐付け済み。福井先生宛の管理者報告メールで全弁護士分の結果を確認
3. 中確度分は各弁護士に本人分のみの確認通知メールが届く。**誤っている場合は当該弁護士がSalesforce上でTaskのWhatIdを正しい基本情報に変更**（またはチャットで「私は○○です。昨日の架電の紐付けを修正して」と起動）
4. 保留分（未登録・判定不能）は手動紐付け：各弁護士がチャットで「私は○○です。昨日の架電の保留分を紐付けて」と起動して対話モードで処理するか、Salesforce上で直接変更（スキルはclaude.ai側のユーザスキルにも同じものを登録しておく）
5. 誤紐付けがあればロールバックCSVから復元

## 留意点

- **Routinesはresearch preview**のため、挙動・制限が変わる可能性があります
- クラウド実行は通常のusage limitsを消費します（日次1回・対象数十件程度なら影響軽微）
- 報告メール（管理者報告・各弁護士宛確認通知とも）はClaudeが福井先生のGmailアカウントから自動送信する形になります
- 失敗時は claude.ai/code/scheduled の実行履歴でセッションログを確認できます
